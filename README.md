# Account Health Early Warning

A working prototype of an AI-assisted account health scan for customer success teams. Synthetic account data is scored by Claude against a versioned scoring prompt, producing a weekly risk tier, reasoning, and a recommended next action with an owner.

Built by Nancy Tu to demonstrate how CS judgment gets encoded into an AI system. The scoring prompt is visible inside the demo.

## What it shows

- Portfolio-level triage. At risk, watch, healthy
- Per-account assessment with the specific signals that drove the tier
- A next action with a named owner, because a warning without an owner is noise
- The full scoring prompt, versioned, so the judgment is auditable

## Stack

Single static HTML file. No build step, no dependencies. Assessments were generated by Claude (claude-sonnet-4-6) from the synthetic telemetry.

## Run locally

Open `index.html` in a browser. That is the whole setup.

## Deploy

Push this repo to GitHub, import it in Vercel, done. Vercel serves `index.html` at the root automatically.

## Tailoring for a specific company

All account names, metrics, labels, and assessment copy live in one marked block at the top of the script section in `index.html`. Swap the usage metric labels and account stories to match the target company's world in about 30 minutes.
